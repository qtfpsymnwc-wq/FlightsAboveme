How Flight Tracking Works
A quick guide to the data you see on FlightsAboveMe

FlightsAboveMe shows aircraft flying near your location using live surveillance broadcasts and aviation data feeds. The goal is to make “what’s overhead?” easy to understand — without needing to be a pilot.

ADS‑B (the main signal)

ADS‑B stands for Automatic Dependent Surveillance–Broadcast. Many aircraft continuously broadcast information like position, altitude, ground speed, and an identifier. Receivers on the ground (and sometimes satellites) pick up these broadcasts and share them through flight tracking networks.

Mode S and MLAT (why trackers can differ)

Not every aircraft transmits the same level of detail all the time. Some transponders report identity (Mode S) but may not include precise GPS position. In those cases, some networks estimate position using MLAT (multilateration), which compares the signal timing across multiple receivers. That’s one reason two sites can show slightly different tracks.

Why a plane might not appear

  • Coverage: receiver density varies by region.
  • Equipment/config: not all aircraft broadcast ADS‑B position.
  • Latency: some feeds are delayed or temporarily missing fields.
  • Filtering: data sources may filter low-quality or incomplete messages.

ICAO24 (the aircraft’s hex ID)

ICAO24 is the 6‑character hexadecimal identifier tied to an aircraft’s transponder. It’s a stable ID that helps services look up aircraft details like registration and type.

Callsign vs flight number

The callsign is what the aircraft uses for air traffic control and surveillance systems (often like DAL123). It usually matches the public flight number, but not always — cargo, charter, and private flights can use different formats.

Altitude, speed, and direction

  • Altitude: often barometric (pressure‑based) altitude, not exact GPS height.
  • Speed: ground speed (wind affects it).
  • Direction: the track over the ground (can differ from “where the nose points”).

How altitude is calculated

Most altitudes shown by flight trackers come from the aircraft’s transponder as barometric altitude (pressure‑based), referenced to standard atmospheric pressure (29.92 inHg / 1013.25 hPa). Air traffic control uses this standardized altitude so all aircraft agree on the same reference even as local weather pressure changes.

  • Barometric vs GPS: baro altitude is not the same as GPS height and can differ by a few hundred feet.
  • MSL vs AGL: trackers usually show altitude above mean sea level (MSL), not height above the ground (AGL).
  • Rounding: some feeds round altitude to 25–100 ft increments depending on the message type.

Squawk codes (transponder codes)

A squawk is a 4‑digit code set on the aircraft’s transponder (Mode A) so radar and ATC systems can identify and manage aircraft. Many squawks are assigned by controllers; some are standard codes used in certain situations.

  • 7700 — general emergency
  • 7600 — radio communications failure
  • 7500 — unlawful interference (hijacking)

Climbing • Descending • Cruising

FlightsAboveMe labels movement using vertical speed (feet per minute): positive = climbing, negative = descending, and near‑zero (or missing) = cruising/level flight.

Our data sources

FlightsAboveMe blends multiple sources to improve reliability and add aircraft details:

  • OpenSky Network — primary live state vectors when available
  • adsb.lol — fallback live aircraft state data
  • AeroDataBox — aircraft enrichment (type/model/operator details)

Important notice

Flight data is provided for informational purposes only and may be delayed, incomplete, or inaccurate. Not for operational or navigation use.
