How Flight Tracking Works

FlightsAboveMe displays aircraft overhead using public surveillance broadcasts and aviation data feeds. Here’s what you’re seeing, where it comes from, and why it can differ from other trackers.

ADS-B in a nutshell

ADS-B (Automatic Dependent Surveillance–Broadcast) is a system where an aircraft broadcasts its position, altitude, speed, and identification, typically derived from onboard GPS. Anyone with the right receiver (or a data feed that aggregates receivers) can pick up these broadcasts.

Coverage varies by region and receiver density, so “what’s overhead” can be missed or delayed in some areas.

ADS-B vs Mode S vs MLAT

  • Mode S: transponder messages that identify the aircraft and can include other data, sometimes without GPS position.
  • ADS-B: broadcasts that include GPS-based position (Mode S ES on 1090 MHz or UAT on 978 MHz in the US).
  • MLAT (multilateration): position estimated by comparing signal arrival times at multiple receivers when ADS-B position isn’t available.

Different sources may use different mixes of ADS-B and MLAT, which can cause small differences between flight tracking sites.

What is ICAO24?

ICAO24 is a unique 6-character hexadecimal identifier assigned to an aircraft’s transponder. It’s like a digital fingerprint used to match live broadcasts to aircraft details (type, registration, operator) when available.

Callsign vs flight number

The callsign is what you’ll see in surveillance/radar data and what pilots use on the radio. It often resembles a flight number (e.g., DAL123), but it can also be a charter/cargo identifier or a format that doesn’t match public schedules.

Altitude, speed, and direction

  • Altitude: often based on barometric pressure (“baro altitude”), not exact GPS height above sea level.
  • Ground speed: speed over the ground (winds can make it differ from airspeed).
  • Track vs heading: track is the direction of motion; heading is where the nose points.

Climbing, descending, and cruising

FlightsAboveMe uses vertical speed (feet per minute) to label the climb state:

  • Positive vertical speed → Climbing
  • Negative vertical speed → Descending
  • Near zero or missing → Cruising (level flight or no reliable vertical speed available)

Where FlightsAboveMe data comes from

FlightsAboveMe blends multiple sources to improve reliability and fill in details:

  • OpenSky Network – primary live state vectors when available
  • adsb.lol – fallback live aircraft state data
  • AeroDataBox – enrichment (aircraft model/type/operator details)

Each source has different coverage and latency. If you see occasional blanks or changes, it’s usually due to upstream availability.

Data limitations

  • Flight tracking data can be delayed, incomplete, or temporarily inaccurate.
  • Coverage depends on aircraft equipment and receiver density.
  • This site is for informational purposes only and is not intended for navigation or operational decisions.
